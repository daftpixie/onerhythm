# Product

This directory is for concise product context that helps contributors make correct implementation decisions.

## Keep here

- Problem framing
- User roles or actors
- Key workflows
- Glossary and domain language
- Public product requirements and constraints

## Avoid here

- Marketing copy
- Speculative roadmap claims
- Deep technical implementation details better suited for architecture docs

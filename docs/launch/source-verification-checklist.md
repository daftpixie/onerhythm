# Source Verification Checklist

Use this checklist for educational, research, evidence, and advocacy pages that reference outside material.

## Required source classes

Allowed source categories:

- peer-reviewed source
- institutional source
- guideline or consensus source
- founder narrative

Founder narrative may support mission or origin copy, but it must not be used as research evidence.

## Verification steps

- Every factual medical-adjacent claim maps to at least one structured source record.
- The source classification is correct.
- The source title, publisher, and URL are accurate.
- The citation metadata is complete enough for a reviewer to verify manually.
- Publish date is recorded if known.
- Review date is recorded if the source is part of the approved corpus.
- Review state is accurate.
- Any summary language has been checked against the source and does not overstate certainty.

## Review questions

- Is this source still current enough for launch use?
- Is the copy accurately framed as a summary rather than a clinical conclusion?
- Does the source support the exact sentence it is attached to?
- Is the difference between evidence and narrative visible to the reader?

## Staleness and update gates

- Source has a visible reviewed date where the content model expects one.
- Source is not past the project’s current review window without explicit note.
- If content is intentionally historical, that is made clear in the page framing.

Launch blocker if:

- a source cannot be verified
- a source link is broken and no verified replacement is available
- a claim is stronger than the source supports
- founder story is being used as evidence for a research claim
